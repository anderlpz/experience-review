Context

Matt built a career timeline site to supplement job applications. It's vibe-coded (AI-assisted), hosted on GitHub Pages, and lives on the develop branch of matthesoundman/experience. The career story is strong: Navy electronics → Apple RF systems → Microsoft driver quality. Each step builds on the last. The timeline format is the right choice for this narrative.

He asked for formatting help and filed 5 issues against himself. He said "don't spend much time on it." This review respects that constraint while identifying what actually matters for his goal.

What's working

The career narrative

Three employers, clear skill escalation, and a throughline (electronics → RF → systems). This is a better story than most resumes tell. The timeline format lets a viewer absorb it in seconds.

Dark theme with frosted nav

The fixed backdrop-filter: blur(18px) nav bar is clean. The overall dark tone is appropriate for a technical/engineering portfolio. The foundation is solid.

The degree band concept

Showing the engineering degree as a connecting element between the Apple card and the timeline (with a dotted connector) communicates "I did this while working full time" visually. Clever structural choice.

Scroll-reveal animations

The IntersectionObserver-based reveals are subtle and appropriate. They add polish without being distracting. Good performance characteristics (GPU-accelerated opacity + transform).

Matt's filed issues

These are the 5 issues Matt filed against himself. All valid, all fixable.

#1
Parallel events display incorrectly
Education that happened concurrently with work doesn't render right. The Technical Schools card (2008–2013, during Navy service) appears sequentially after Microsoft instead of alongside Navy.
#2
Card symbols aren't great
Needs the Navy ET (Electronics Technician) rating symbol and a proper Microsoft icon (Windows logo or driver-related). Current icons: obscure Unicode, Apple logo (only renders on Apple devices), empty square, triangle.
#3
Technical schools card is too big
Should be subtle like the degree band, not a full-size card. Matt wants it "smaller and set in parallel." The .compact CSS class is applied in HTML but has no styles defined — it was intended but never implemented.
#4
Website header needs review
"The header needs review and refinement. It does not serve well at the moment." The hero takes 92vh before any timeline content appears.
#5
LinkedIn ID is missing
The placeholder YOUR_PROFILE is still in the LinkedIn href. Quick fix once Matt drops in his actual LinkedIn slug.

AI design fingerprints

This is the critical finding. The site currently reads as AI-generated within seconds. For a job application supplement, this undermines the implicit message ("I care about quality"). Here's what flags it:

Detected

Rainbow accent palette. Five competing colors (blue, purple, green, cyan, yellow). Design principle: one accent color. Multiple accent colors is the most common AI palette signature.

Detected

Neon outer glows. box-shadow: 0 0 38px on timeline dots. Classic AI glow effect. Subtle inner borders or tinted shadows read as human-designed.

Detected

Identical card structure. Every card has the exact same proportions, same layout, same icon treatment. Human designers vary cards by type and importance.

Detected

CSS grid overlay. The body::before 36px grid pattern is a common AI dark-theme cliche. Adds visual noise without serving the content.

Detected

Oversized centered hero. 92vh hero with huge H1 and tight letter-spacing. Textbook AI landing page. The hero should earn its space or get out of the way.

Detected

Generic Unicode icons. Arbitrary Unicode characters as icons rather than purpose-made SVGs or a proper icon set. The Apple logo character only renders on Apple devices — Windows recruiters see a broken glyph.

Through the recruiter lens

Matt's audience is hiring managers and recruiters. They'll typically arrive on mobile (LinkedIn → click link → phone browser), spend 10–20 seconds, and want to immediately grasp the career trajectory.

92vh hero pushes content below the fold

The actual career timeline — the reason someone clicks the link — is invisible until they scroll past a near-full-screen hero. On mobile, this is especially costly. The hero should introduce, not gate.

Navigation disappears on mobile

Below 850px, both navigation links are hidden. Only the brand name survives. The "Path" and "Resume" links vanish exactly when they matter most — on the device recruiters are actually using.

No Open Graph meta tags

When this URL is shared on LinkedIn or pasted into an application, the link preview will be generic. No title, no description, no image. First impressions matter — the meta tags are what the recruiter sees before they click.

Cross-platform icon rendering

The Apple card uses a Unicode character that only renders on macOS/iOS. On a Windows recruiter's browser, it shows as a missing character glyph. SVGs solve this universally.

Recommendations

Organized by what Matt asked for versus what would make the biggest difference for his goal.

Tier 1 — What he asked for (his 5 issues)

Quick formatting fixes, icon swaps, parallel layout for concurrent events, LinkedIn URL, header rethink. These are all straightforward CSS/HTML changes. The parallel events problem (#1, #3) is the most structural — it needs a layout approach for showing overlapping timelines (e.g., a split-column or side-branch pattern for education alongside work).

Tier 2 — What makes it actually land

Palette discipline: Pick one accent color (the blue is strong enough on its own), use brightness/saturation variants for the other employers instead of entirely different hues.

De-AI the visual language: Remove grid overlay, replace neon glows with subtle borders, vary card proportions by importance, use proper SVG icons.

Mobile-first hero: Shrink the hero to ~50vh or less, or restructure it so the timeline is partially visible on first load. The "Resume PDF" button should be immediately reachable.

OG meta tags: 5 lines of HTML that transform how the link looks when shared. This is the lowest-effort, highest-impact change.

Favicon: A missing favicon is a small thing that signals incompleteness.
